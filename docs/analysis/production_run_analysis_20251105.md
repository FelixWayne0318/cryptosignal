## ⚠️ 四、发现的异常和问题

### 4.1 T因子异常：几乎全是±100

**观察**：
```
MANTAUSDT: T=+100
APEUSDT:   T=+100
ADAUSDT:   T=+100
HOOKUSDT:  T=+100
2ZUSDT:    T=+100
...
```

**问题**：
- T因子几乎全部是+100或-100
- 这不正常，T因子应该有更多中间值

**可能原因**：
1. T因子归一化过于敏感，容易触及极值
2. 当前市场确实是极端单边趋势（牛市）
3. T因子计算公式可能需要调整

**建议**：
- 检查T因子的tanh scale参数
- 统计T因子的历史分布
- 考虑使用更平滑的映射函数

### 4.2 F因子异常：几乎全是-100

**观察**：
```
MANTAUSDT: F=-100
APEUSDT:   F=-18
HOOKUSDT:  F=-100
大多数:     F=-99 或 F=-100
```

**问题**：
- F因子（Funding Rate）几乎全部极端负值
- 这可能反映市场过热（多头支付空头）

**可能原因**：
1. 当前市场Funding Rate确实普遍为负（牛市特征）
2. F因子归一化可能过于敏感
3. 需要重新校准F因子的归一化基准

**建议**：
- 检查实际Funding Rate数据
- 调整F因子的归一化方法
- 考虑使用滚动窗口而非固定阈值

### 4.3 软约束警告频繁

**观察**：
```
162个信号中，大多数显示：
  软约束: ⚠️ P=0.xxx<0.7xx
```

**这是正常的吗？**

✅ **是的，这是防抖动系统正常工作**

**解释**：
1. P值略低于动态阈值，但通过了Prime强度门槛
2. 防抖动系统要求这些信号等待确认（K/N=1/2）
3. 如果信号持续，下一轮会进入Prime发布

**这说明**：
- 当前市场处于"普遍多头但不够极端"的状态
- T=100拉高了基础分，但C/M/O不够强
- 系统正确地保守处理边缘信号

## 📈 五、C/M/O因子组合模式分析

### 5.1 强多头模式（C+M+O全正）

**特征**: C>0, M>0, O>0
**案例**: APEUSDT, MANTAUSDT, KAIAUSDT
**解释**: 资金流入 + 动量确认 + 持仓增长
**质量**: ⭐⭐⭐⭐⭐ 最高质量

### 5.2 价涨量跌模式（M+, C-, O-）

**特征**: M>>0, C<0, O<0
**案例**: 2ZUSDT (M=99, C=-51, O=-29)
**解释**: 价格暴涨但资金流出、持仓减少
**质量**: ⚠️⚠️ 不可持续，可能是假突破或空头回补

### 5.3 空头增仓模式（T+, C-, M-, O+）

**特征**: T>0, M<0, C<0, O>0
**案例**: ADAUSDT (T=100, M=-64, C=-89, O=80)
**解释**: 趋势向上但资金流出、动量负、持仓增
**质量**: ⚠️⚠️⚠️ 需反向理解，可能是空头建仓

### 5.4 弱多头模式（T+, 部分因子正）

**特征**: T>0, 但C/M/O分歧
**案例**: HOOKUSDT, GOATUSDT
**解释**: 趋势多头但缺乏全面支持
**质量**: ⚠️ 质量较低，需谨慎
